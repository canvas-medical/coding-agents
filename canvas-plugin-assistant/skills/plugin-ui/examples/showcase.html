Multiple dates

selection-mode="multiple" tracks a chronological set of dates as the value (comma separated ISO keys). The optional picked-list attribute renders a built in column with a Clear all link and per row remove buttons. Default is no list (the day grid alone communicates the selection). Set picked-list="right" when there is room beside the calendar, or picked-list="below" for narrower contexts. Pattern for recurring appointments, medication adherence views, days a service was rendered.

Multiple dates, week granularity

multi-add-granularity on top of selection-mode="multiple" expands a single click into a set of dates. Value workweek expands to Monday through Friday of the clicked date's week. Value week expands to all seven days, respecting the week-start attribute. Set toggle semantics, click any day in a fully selected set to remove the set, click any day in a partial or empty set to add the missing dates. Disabled dates inside the set are skipped without aborting the rest. Add multi-add-exclusive when only one set should be selected at a time, the click replaces the current selection and clicking the same set again clears it. The picked-list attribute is independent of granularity, every variant supports it. Pattern for shift planning, weekly availability blocks, and recurring clinical protocols.

multi-add-granularity="workweek"
multi-add-granularity="week" with picked-list="right"
multi-add-granularity="workweek" with multi-add-exclusive and picked-list="below"

Date input dropdown

canvas-date-input wraps canvas-calendar in a combobox style trigger. The closed state looks identical to canvas-dropdown and canvas-combobox, so it slots into form rows without breaking row height cohesion. Click the field or press ArrowDown, Enter, or Space to open the panel. The open panel shares the blue active border on three sides, with a thin gray divider between the input row and the calendar grid. The panel flips above the input when there is no room below. Selecting a date fills the field and closes the panel. Forwards min, max, disabled-dates, week-start, size, disabled, error, name, label, placeholder, required, and the form value is the ISO date string.

Try the third field, only May through August 2026 are selectable. Try opening near the bottom of the viewport to see the upward flip.

Status legend

Status legend
